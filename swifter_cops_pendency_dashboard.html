DB Classification Logic

Current working logic for review
FM PendencyShipment is manifested but Network Arrival Date is blank. This is pickup/FM pendency and is not eligible for MM/LM calculation yet.
MM PendencyNetwork Arrival Date is present, but LM Arrival Time is blank and transit aging is within 2 days.
Short ReceivedIn Transit Time exists, LM Arrival Time is blank, and aging is more than 2 days.
LM PendencyNetwork Arrival Date and LM Arrival Time are present, but shipment is not completed, COD completed, or RTO delivered.
Promise Date ViewUses Delivery Slot End Timestamp as customer promise date and shows promise buckets. Rows with blank Network Arrival Date are excluded.
Lane CountsManifested in the lane view means non-terminal shipments only. Pending means manifested but Network Arrival Date is blank.
LaneForward lane is FM Station Name - LM Station Name. RTO lane is LM Station Name - FM Station Name. Lane never uses current station.
AgingForward/RTO aging views use overall shipment aging, prioritising Network Arrival Date when present. Buckets are <=2, 3-5, 5-7, 7-10, 10-15, >15 days.
DirectionOrder Type = Forward is forward pendency. Order Type = RTO is reverse pendency.
COD PendencyCOD rows are tracked only when Payment Method = cod/cash. Delivered means cash pending at hub; COD Deposited means deposited in portal; COD Reconciled means COPS/payment verified; Pending COD Clearance and Pending COD Verification remain COPS/payment action buckets.
Access RulesViewer users can see pendency only. Admin users can upload files, open the COPS control panel, and update RCA/description at TID level.
RCA RulesLost requires Employee Facility ID. Tech Issues requires issue description. RCA is stored locally against AWB/ID and can be exported as CSV by admin.
